Section 1: High-Level Architecture

Modular monolith with background workers — "PromptBase"

System Architecture
🖥️ FRONTEND (React + Vite)
• Chat Interface (SSE streaming)
• Task Forms (structured input)
• Document Upload
• Admin Dashboard
• Team Management
▼ HTTP / SSE ▼
⚡ FASTAPI APPLICATION
Auth Module
JWT, roles, teams
Chat Module
Conversations, SSE
Prompt Compiler
Pack loading, routing
Admin Module
Packs, teams, config
🔄 BACKGROUND WORKERS
• Document parsing (PyMuPDF, pdfplumber)
• Text chunking & embedding
• OCR microservice calls
• Vector store indexing
🤖 LLM PROVIDER LAYER
• Custom abstraction (no LiteLLM)
• Anthropic SDK → Claude
• OpenAI SDK → GPT models
• httpx → OpenRouter, Ollama
• Per-team model config
🗄️ DATA LAYER
• PostgreSQL (users, teams, chats, configs)
• pgvector (document embeddings)
• Redis (task queue, session cache)
• File storage (uploaded docs, prompt packs)
🐳 Docker Compose
web (React) → api (FastAPI) → worker (Celery) → db (Postgres+pgvector) → redis → file-storage

Key Design Decisions

  • Prompt Compiler is an internal module — loads packs from DB/filesystem, classifies requests, selects domain modules, compiles system prompt
  • Background Workers (Celery + Redis) handle document processing so chat stays responsive
  • LLM Provider Layer uses official SDKs directly — one interface, multiple backends
  • File Storage is abstracted — local disk for dev, S3-compatible for production
  • Each team gets: their own prompt pack, model config, user list, conversation history, document store
